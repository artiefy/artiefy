# Next.js Agent Skills

[](https://github.com/vercel-labs/next-skills?tab=readme-ov-file#nextjs-agent-skills)

Agent skills for common Next.js workflows.

## Essential Skills

[](https://github.com/vercel-labs/next-skills?tab=readme-ov-file#essential-skills)

Start here. These background skills are auto-applied to prevent common mistakes.

### `next-best-practices`

[](https://github.com/vercel-labs/next-skills?tab=readme-ov-file#next-best-practices)

Core Next.js knowledge:

- [File Conventions](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/file-conventions.md) - Project structure and special files
- [RSC Boundaries](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/rsc-boundaries.md) - Server/Client Component rules
- [Data Patterns](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/data-patterns.md) - Fetching and mutation strategies
- [Async Patterns](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/async-patterns.md) - Next.js 15+ async APIs
- [Directives](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/directives.md) - `'use client'`, `'use server'`, `'use cache'`
- [Functions](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/functions.md) - Navigation hooks, server functions, generate functions
- [Runtime Selection](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/runtime-selection.md) - Node.js vs Edge runtime
- [Error Handling](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/error-handling.md) - Error boundaries and redirects
- [Route Handlers](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/route-handlers.md) - API routes with `route.ts`
- [Metadata](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/metadata.md) - SEO, OG images, sitemaps
- [Image](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/image.md) - `next/image` optimization
- [Font](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/font.md) - `next/font` optimization
- [Bundling](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/bundling.md) - Package compatibility, CSS imports, polyfills
- [Scripts](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/scripts.md) - Third-party scripts, Google Analytics
- [Hydration Errors](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/hydration-error.md) - Debugging mismatches
- [Suspense Boundaries](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/suspense-boundaries.md) - CSR bailout, Cache Components requirements
- [Parallel Routes](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/parallel-routes.md) - Modal patterns with intercepting routes
- [Self-Hosting](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/self-hosting.md) - Docker, standalone output, ISR
- [Debug Tricks](https://github.com/vercel-labs/next-skills/blob/main/skills/next-best-practices/debug-tricks.md) - MCP endpoint, rebuild specific routes

## Contributing

[](https://github.com/vercel-labs/next-skills?tab=readme-ov-file#contributing)

Each skill follows the [Agent Skills open standard](https://github.com/anthropics/skills):

1.  Create a directory under `skills/` with the skill name (prefix with `next-`)
2.  Add a `SKILL.md` file with YAML frontmatter:

## undefinedyaml

name: next-skill-name
description: Brief description
user-invocable: false # for background skills

---

undefined

3.  For complex skills, add additional `.md` files and reference them from `SKILL.md`
